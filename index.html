简述 Java 的运行机制。
Java 程序运行时，必须经过编译和运行两个步骤。首先将扩展名为 java 的源文件进行编译，生成扩展名为 class 的字节码文件，然后 java 虚拟机将字节码文件进行解释执行，并将结果显示出来。
02
什么是构造方法？Java 中构造方法须同时满足的条件。
构造方法是类的一个特殊成员，它会在类实例化对象时被自动调用。
构造方法须同时满足的条件为：方法名和类名相同；在方法名的前面没有返回值类型的声明；在方法中不能使用 return 语句返回一个值，但是可以单独写 return 语句来作为方法的结束。
03
什么是方法的重写？什么是方法的重载？两者有什么区别？
在继承关系中，子类中需要对继承的方法进行一些修改，即为方法的重写。在一个程序中定义多个名称相同，但是参数的类型或个数不同的方法，即为方法的重载。
区别：发生位置不同（重载在类中，重写在继承关系中）；参数列表限制不同（重载必须不同，重写必须相同）；返回值类型（重载无关，重写必须一致）；访问权限（重载无关，重写子类方法权限不能小于父类方法权限）。

什么是自动装箱和自动拆箱？
自动装箱是指将基本数据类型的变量赋给对应的包装类变量；自动拆箱是指将包装类对象类型直接赋给一个对应的基本数据类型变量。

简述什么是集合？并列举集合中常用的类和接口。
集合像一个容器，专门用来存储 Java 对象（实际上是对象的引用），这些对象可以是任意的数据类型，并且长度可变。集合分为单列集合 Collection 和双列集合 Map。Collection 有两个子接口 List 和 Set。Map 接口的主要实现类有 HashMap 和 TreeMap。

什么是流？Java 语言中分为哪两种流？这两种流有何区别？
在 Java 中，将通过不同输入输出设备（键盘、内存、显示器、网络等）之间的数据传输抽象表述为流。
按操作数据单位分：字节流（以字节为单位读写）和字符流（以字符为单位读写）。
按传输方向分：输入流（只能读取数据）和输出流（只能写入数据）。
按功能分：节点流（直接连接数据源）和处理流（封装节点流）。

简述 JDBC 中 Driver、Connection、Statement 和 ResultSet 的功能。
Driver 接口是所有 JDBC 驱动程序必须实现的接口，专门提供给数据库厂商使用。Connection 接口代表 Java 程序和数据库的连接对象，只有获得该连接对象后，才能访问数据库并操作数据表。Statement 接口用于执行静态的 SQL 语句，并返回一个结果对象。ResultSet 接口用于保存 JDBC 执行查询时返回的结果集，该结果集封装在一个逻辑表格中。

什么是多态？
多态意味着一个对象有着多种形态，可以在特定的情况下表现不同的状态，从而对应着不同的属性和方法。简单的说，多态就是使用父类类型的变量引用子类对象，根据被引用子类对象的特性，程序会得到不同的运行效果。

简述什么是抽象类和什么是接口？
在 Java 中，使用 abstract 关键字修饰的类称之为抽象类。抽象类是不能被实例化的，通常需要写一个子类来继承抽象类，同时实例化子类来获得该类的对象。抽象类通常用于表示一种抽象的概念。接口可以说是一种特殊的抽象类，接口中只能定义常量、抽象方法、静态方法和默认方法。要实现接口需要使用 interface 关键字。

简述字节流和字符流的区别？
字节流的两个基类是 InputStream 和 OutputStream，字符流的两个基类是 Reader 和 Writer，它们都是 Object 类的直接子类。字节流是处理以 8 位字节为基本单位的字节流类；Reader 和 Writer 类是专门处理 16 位字节的字符流类。

简述 JDBC 编程步骤。
（1）加载数据库驱动；（2）通过 DriverManager 获取数据库连接；（3）通过 Connection 对象获取 Statement 对象；（4）使用 Statement 执行 SQL 语句；（5）操作 ResultSet 结果集；（6）关闭连接，释放资源。

简述 TCP 协议和 UDP 协议。
UDP 协议是无连接通信协议，数据的发送端和接收端不建立逻辑连接。UDP 协议消耗资源小，通信效率高，通常用于音频、视频和普通数据的传输，但不能保证数据的完整性。TCP 协议是面向连接的通信协议，传输数据前先在发送端和接收端建立逻辑连接，提供了可靠无差错的数据传输。TCP 连接中必须明确客户端与服务器端，每次连接的创建都需要经过“三次握手”。

简述 Java 应用程序与 Java 小程序的区别？
运行环境不同：Java 应用程序可独立运行于计算机上，Java 小程序需要借助 Web 浏览器才能运行。功能不同：Java 应用程序功能相对更强大，可用于各种复杂的任务，Java 小程序通常扮演更为辅助性的角色。复杂度不同：Java 应用程序更复杂、更庞大，包含的代码也更多。运行方式不同：Java 应用程序可以通过命令行、脚本等方式运行，Java 小程序只能通过 Web 浏览器运行。

简述基本类型变量和引用类型变量的区别（举例说明）。
基本类型变量：使用 byte、short、int、long、float、double、boolean、char 这八种类型声明出来的变量。引用类型变量：使用任意一个类、任意一个接口、任意一个数组所声明出来的变量，引用类型变量接收的只能是对象。例如：Student s = new Student(); 区别：基本数据类型在声明时系统就给它分配空间，引用则不同，它声明时只给变量分配了引用空间，而不分配数据空间。

简述方法的重载与覆盖的区别。
方法重载指在一个类中创建了多个方法，具有相同的方法名，但参数的个数不同、参数的数据类型不同、参数顺序不同。覆盖是指子类中定义的方法与父类中方法名相同，子类在调用这个同名方法时，默认调用它自己定义的方法，而将从父类那里继承来的方法“覆盖”住。重载是在一个类中有多个方法，方法名相同而参数不同；覆盖是指父类和子类中方法同名且拥有相同的参数列表，子类在调用时默认调用子类的同名方法而覆盖掉父类的方法。

什么是接口？为什么要定义接口？
接口是一种用来定义程序的协议，它描述可属于任何类或结构的一组相关行为。接口的意义在于抽象、不拘细节，从而使同类事物在同一高度具有通用及可替代性。定义接口与定义类非常相似，接口是由常量和抽象方法组成的特殊类。一个类只能由一个父类，但是它可以同时实现若干个接口，即实现了多重继承。

InputStream、OutputStream、Reader 和 Writer 四个类的区别？
OutputStream 和 InputStream 是字节输入和输出流。Reader 和 Writer 是字符流。字节流读写二进制文件，字符流读写文本文件。

什么是平台无关性？JAVA 语言是如何实现平台无关性的？
Java 是与平台无关的语言，主要指用 Java 编写的应用程序不用修改就可在不同的软硬件平台上运行。Java 语言是靠 Java 虚拟机（JVM）在目标代码级实现平台无关性的。Java 源程序通过编译器会被转换成与平台无关的字节码，字节码是 JVM 的指令组，任何一种能够运行 Java 字节码的软件均可看做是 JVM。

自动类型转换的前提是什么？简述转换时的优先级顺序。
前提：转换前的数据类型与转换后的数据类型兼容；转换后的数据类型的表示范围比转换前的数据类型的表示范围大。优先级顺序从低到高：byte, short, char, int, long, float, double。

简述 JAVA 程序中的成员变量和局部变量的区别。
1）定义的位置不一样：成员变量在类体内部方法体外面，局部变量在方法体内部。2）作用范围不一样：成员变量作用范围是整个类，局部变量作用范围是方法体。3）默认值不一样：成员变量有默认值，局部变量没有默认值。4）内存的位置不一样：成员变量位于堆内存，局部变量位于栈内存。5）生命周期不一样：局部变量随着方法进栈而诞生、出栈而消亡，成员变量随着对象创建而诞生、GC 回收而消亡。

对象的相等，比的是内存中存放的内容是否相等。而引用相等，比较的是它们指向的内存地址是否相等。

什么是接口？简述接口与抽象类的区别。
接口是用于完成特定功能的若干属性组织成相对独立的属性集合。定义接口与定义类非常相似，接口是由常量和抽象方法组成的特殊类。一个类只能由一个父类，但可以同时实现若干个接口，实现了多重继承。与类的继承不同的是，一个接口可以有一个以上的父接口，新接口将继承所有父接口中的属性和方法。

Java 实现多线程有哪两种途径？有什么区别？
实现多线程的方法有两种：一种是继承 java.lang 包中的 Thread 类；另一种是用户在定义自己的类中实现 Runnable 接口。直接继承 Thread 类的特点是编写简单，可以直接操纵线程，但缺点是若继承 Thread 类，就不能再继承其他类。使用 Runnable 接口的特点是可以将 Thread 类与所要处理的任务的类分开，形成清晰的模型，还可以从其他类继承，从而实现多重继承的功能。
