1. Java 运行机制与环境基础
01
简述 Java 的运行机制。
Java 程序运行时，必须经过编译和运行两个步骤：
1. 编译：首先将扩展名为 .java 的源文件进行编译，生成扩展名为 .class 的字节码文件。
2. 运行：然后 Java 虚拟机（JVM）将字节码文件进行解释执行，并将结果显示出来。
02
什么是平台无关性？JAVA 语言是如何实现平台无关性的？
平台无关性是指用 Java 编写的应用程序不用修改就可在不同的软硬件平台上运行。Java 语言是靠 Java 虚拟机（JVM）在目标代码级实现平台无关性的。Java 源程序通过编译器会被转换成与平台无关的字节码（JVM 的指令组）。任何一种能够运行 Java 字节码的软件均可看作是 JVM。
03
简述 JDK 与 JRE 的关系。
JRE（Java Runtime Environment）：包含了 Java 虚拟机和 Java 基础类库。它是运行 Java 程序所需要的软件环境，面向想运行 Java 程序的用户。
JDK（Java Development Kit）：是程序员编写 Java 程序所需的开发工具包。
关系：运行 Java 程序只需安装 JRE；若需要编写或开发 Java 程序，则必须安装 JDK。
04
简述 Java 应用程序与 Java 小程序的区别。
两者主要在以下四个方面存在差异：
运行环境：应用程序可独立运行于计算机；小程序需要借助 Web 浏览器运行。
功能：应用程序功能更强大，可处理复杂任务；小程序通常扮演辅助性角色。
复杂度：应用程序更复杂、庞大，包含的代码更多。
运行方式：应用程序可通过命令行、脚本等多种方式运行；小程序只能通过 Web 浏览器运行。
2. Java 面向对象（OOP）核心概念
05
什么是构造方法？Java 中构造方法须同时满足的条件。
构造方法是类的一个特殊成员，在类实例化对象时被自动调用。须同时满足以下三个条件：
1. 方法名和类名相同。
2. 方法名前面没有返回值类型的声明。
3. 方法中不能使用 return 语句返回一个值，但是可以单独写 return 语句来作为方法的结束。
06
什么是方法的重写？什么是方法的重载？两者有什么区别？
方法的重写（Overriding）：在继承关系中，子类需要对继承自父类的方法进行修改。
方法的重载（Overloading）：在一个程序中定义多个名称相同，但参数类型或个数不同的方法。
主要区别：
位置：重载发生在本类中；重写发生在继承关系的子类中。
参数列表：重载必须不同；重写必须相同。
返回值类型：重载与返回值无关；重写必须一致。
访问权限：重载与权限无关；重写中子类的权限不能小于父类。
07
简述方法的重载与覆盖的区别。
重载：是在一个类中有多个方法，方法名相同而参数不同（个数、类型或顺序不同）。
覆盖（即重写）：是指父类和子类中方法同名且拥有相同的参数列表。子类在调用时默认调用子类的同名方法而覆盖掉父类的方法。如果一定要调用父类的同名方法，可以使用 super 关键字来指定。
08
什么是多态？
多态意味着一个对象有着多种形态，可以在特定情况下表现不同状态，从而对应不同的属性和方法。简单来说，多态就是使用父类类型的变量引用子类对象，根据被引用子类对象的特性，程序会得到不同的运行效果。
09
简述什么是抽象类和什么是接口？
抽象类：使用 abstract 关键字修饰的类。它不能被实例化，通常需要写一个子类来继承并实例化以获得对象，常用于表示一种抽象概念。
接口：可以说是一种特殊的抽象类，只能定义常量、抽象方法、静态方法和默认方法。使用 interface 关键字定义。
10
什么是接口？为什么要定义接口？
接口是一种用来定义程序的协议，描述了可属于任何类或结构的一组相关行为。定义意义：
1. 抽象化：使同类事物在同一高度具有通用性和可替代性。
2. 多重继承：Java 类只能有一个父类，但通过实现多个接口，实际上就获得了多个父类的特性，从而实现了多重继承的功能。
11
什么是接口？简述接口与抽象类的区别。
接口是用于完成特定功能的若干属性组织成的相对独立的属性集合。主要区别特征：
组成：接口是由常量和抽象方法组成的特殊类。
访问控制：接口的访问控制符只有 public 一个。
继承特性：一个接口可以有一个以上的父接口（用逗号分隔形成父接口列表），新接口将继承所有父接口的属性和方法。
3. 变量类型、数据转换与内存特性
12
简述基本类型变量和引用类型变量的区别（举例说明）。
基本类型变量：包括 byte, short, int, long, float, double, boolean, char。系统在声明时即为其分配空间。
引用类型变量：指使用类、接口、数组声明的变量。引用类型变量接收的只能是对象，例如 Student s = new Student();
核心区别：基本数据类型在声明时分配空间；引用变量在声明时只分配引用空间，而不分配数据空间。
13
自动类型转换的前提是什么？简述转换时的优先级顺序。
前提条件：
1. 转换前后的数据类型必须兼容。
2. 转换后的数据类型表示范围比转换前的表示范围大。
优先级（由低到高）：byte → short, char → int → long → float → double。
14
什么是自动装箱和自动拆箱？
自动装箱：指将基本数据类型的变量直接赋给对应的包装类变量。
自动拆箱：指将包装类对象直接赋给一个对应的基本数据类型变量。
15
简述 JAVA 程序中的成员变量和局部变量的区别。
维度 成员变量 局部变量
定义位置 类体内部，方法体外面 方法体内部
作用范围 整个类 方法体内部
默认值 有默认值 没有默认值
内存位置 位于堆内存 位于栈内存
生命周期 随着对象创建而诞生，GC 回收而消亡 随着方法进栈而诞生，出栈而消亡
16
简述 JAVA 程序中对象的相等和指向它们的引用相等的区别。
对象相等：比较的是内存中存放的内容是否相等。
引用相等：比较的是它们指向的内存地址是否相等。
4. Java 核心库：集合与 IO 流
17
简述什么是集合？并列举集合中常用的类和接口。
集合是一个专门用来存储 Java 对象（对象引用）的容器，长度可变，可存储任意数据类型。
单列集合（Collection）：子接口包括 List 和 Set。
双列集合（Map）：主要实现类有 HashMap 和 TreeMap。
18
什么是流？Java 语言中分为哪两种流？这两种流有何区别？
流是对不同设备间数据传输的抽象表述。
1. 按数据单位分：字节流（以字节为单位读写）；字符流（以字符为单位读写）。
2. 按传输方向分：输入流（只能读取数据）；输出流（只能写入数据）。
3. 按功能分：节点流（直接连接数据源）；处理流（封装已存在的节点流以增强读写能力）。
19
简述字节流和字符流的区别？
基类不同：字节流基类是 InputStream / OutputStream；字符流基类是 Reader / Writer。
处理单位：字节流处理 8 位字节；字符流（Reader/Writer 类）专门处理 16 位字符。
20
InputStream、OutputStream、Reader 和 Writer 四个类的区别？
InputStream 和 OutputStream 为字节流，主要用于读写二进制文件。
Reader 和 Writer 为字符流，主要用于读写文本文件。
5. Java 高级特性
21
简述 JDBC 中 Driver、Connection、Statement 和 ResultSet 的功能。
Driver：JDBC 驱动程序必须实现的接口，供数据库厂商使用。
Connection：代表 Java 程序与数据库的连接对象，是访问数据库的前提。
Statement：用于执行静态 SQL 语句并返回结果对象。
ResultSet：用于保存查询返回的结果集，并将其封装在逻辑表格中。
22
简述 JDBC 编程步骤。
1. 加载数据库驱动。
2. 通过 DriverManager 获取数据库连接。
3. 通过 Connection 对象获取 Statement 对象。
4. 使用 Statement 执行 SQL 语句。
5. 操作 ResultSet 结果集。
6. 关闭连接，释放资源。
23
用 Java 程序实现流式 Socket 通信，需要使用哪两个类？它们是如何定义的？
需要使用 ServerSocket 类和 Socket 类。定义方式如下：
服务端：ServerSocket serverSocket = new ServerSocket(7788);
客户端：Socket socket = new Socket(InetAddress.getLocalHost(), 7788);
24
简述 TCP 协议和 UDP 协议。
UDP 协议：无连接通信协议。消耗资源小，效率高，常用于音视频传输；但不能保证数据的完整性。
TCP 协议：面向连接的通信协议。提供可靠无差错的传输，连接的创建需要经过"三次握手"。
25
Java 实现多线程有哪两种途径？有什么区别？
1. 继承 Thread 类：编写简单，可直接操纵线程；但子类无法再继承其他类。
2. 实现 Runnable 接口：将任务与线程分离，模型清晰；且支持类继承其他类，从而实现多重继承功能。
